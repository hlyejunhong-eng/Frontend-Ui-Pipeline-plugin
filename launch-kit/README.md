# Frontend UI Pipeline 自媒体引流包

这套内容用于把插件讲清楚、讲动人，并让用户从 GitHub 下载后能立刻开始试。

## 你可以直接发布的材料

- 安装到实用说明：[`install-to-use.zh-CN.md`](install-to-use.zh-CN.md)
- 三阶段流程介绍：[`workflow.zh-CN.md`](workflow.zh-CN.md)
- 可发布案例稿：[`case-draftpilot.zh-CN.md`](case-draftpilot.zh-CN.md)
- 自媒体脚本和标题：[`creator-posts.zh-CN.md`](creator-posts.zh-CN.md)
- 视觉海报源文件：[`social-assets/index.html`](social-assets/index.html)
- 导出的 PNG 海报：[`cover-wide.png`](social-assets/export/cover-wide.png)、[`case-square.png`](social-assets/export/case-square.png)、[`story-vertical.png`](social-assets/export/story-vertical.png)

## 主传播钩子

不会画 UI、不会写前端，也能把一个普通页面变成高端定制 UI，并且不是停在概念图，而是走到组件资产、动效规则和真实可运行前端。

更具体地说：用户安装的是 Codex 插件，打开的是 Codex 应用里的新线程，在底部消息输入框发送旧页面截图、项目路径、localhost 地址、Figma 链接或页面描述，再粘贴启动 prompt。

## 推荐发布顺序

1. 发第一条：before/after 冲击图，标题打痛点。
2. 发第二条：三阶段流程，强调不是普通 AI 生图。
3. 发第三条：DraftPilot 案例，展示真实截图和组件库。
4. 评论区置顶 GitHub 链接和安装命令。
5. 邀请用户在 Codex 新线程里粘贴一个旧页面截图或项目路径，做下一条公开视频案例。

## 图片怎么用

- `cover-wide.png`：B 站、YouTube、视频号横版封面。
- `case-square.png`：小红书、即刻、朋友圈图文首图。
- `story-vertical.png`：抖音、视频号、快手竖版封面。
