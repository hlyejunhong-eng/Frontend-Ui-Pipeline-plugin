# 三阶段流程介绍

## 一句话版本

先把风格锁死，再把资产做全，最后落到真实前端。

## 阶段一：UI Ideation

核心问题：这个页面应该长成什么样，为什么高级，怎么被实现？

输入：

- 旧页面、截图、Figma、本地项目、页面描述

输出：

- 高端定制方向
- 桌面/移动预览
- 像素级 UI brief
- 阶段二生成指南

重点：

- 不只写“现代、简洁、高级”
- 要写清楚布局、背景、按钮、文字、状态、动效
- 要写清楚插画分层和可调参数

## 阶段二：Asset Production

核心问题：阶段三实现时到底调用哪些资产和组件？

输出：

- 背景/插图/遮罩/图标
- 完整基础组件资产包
- 组件预览页
- asset manifest
- asset handoff

重点：

- 不只生成当前页面用到的东西
- 基础按钮、角标、卡片、组合框、图标、导航、通告、搜索、标题、弹窗、动画都要有
- 用户必须审核通过

## 阶段三：Frontend Implementation

核心问题：它能不能真实跑起来？

输出：

- 真实前端代码
- 接口接入或 mock 数据
- 桌面/移动截图
- 验证命令和本地运行说明

重点：

- 不能只停在设计图
- 没有 API 就用同 shape mock
- 没有项目就生成独立可运行页面

## 适合发给用户的解释

普通 AI UI 生成器通常只给你一张图。Frontend UI Pipeline 把中间最难的交付链路补上：风格锁定、组件资产、动效规则、真实代码、截图验证。
