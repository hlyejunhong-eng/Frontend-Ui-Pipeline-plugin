# Launch Playbook

Use this checklist to move the repository toward broad GitHub adoption.

## Positioning

- Audience: founders, product managers, indie builders, and engineers who want premium UI but do not have a designer or frontend specialist available.
- Promise: go from existing app flow to design brief, real art assets, and shippable frontend implementation.
- Differentiator: the pipeline does not stop at a screenshot; it forces asset approval and real code integration.

## Demo Checklist

For each public demo:

1. Start from a real existing app screen.
2. Show the original screen.
3. Run `$frontend-ui-ideation` and show the generated preview plus `phase1-ui-brief.md`.
4. Run `$frontend-asset-production` and show the approved asset contact sheet.
5. Run `$frontend-implementation` and show the local app before and after.
6. Publish screenshots, the prompt, and the repo commit.

## First Ten Issues To Welcome

- More example prompts for SaaS dashboards.
- More example prompts for consumer mobile screens.
- Asset naming improvements.
- Motion handoff examples.
- Integration notes for React, Vue, Next.js, Nuxt, and plain HTML.
- Better acceptance checklists for Phase 2.
- Better mock-data conventions for Phase 3.
- A gallery of before/after examples.
- A short video demo script.
- Real user reports where the workflow fails.

## Growth Loop

- Keep install under five minutes.
- Add one strong before/after example per week.
- Turn every repeated support question into README copy.
- Ask early users for screenshots and anonymized case studies.
- Keep GitHub topics current so the repo appears in relevant searches.
- Prefer small, visible improvements over hidden internal complexity.
